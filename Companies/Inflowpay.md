## Business Description

- **Problem:** Existing payment rails (Stripe, PayPal) serve human flows; agent-native commerce requires a new payment layer where agents can register, pay, and purchase end-to-end under programmable guardrails.
- **Product:** Agent-native payment system — policy engine as system spine (MD file/context for ML models), MCP integration for Claude, Cursor, and other AI platforms. Complete rebuild from scratch (not a 0.5 solution cobbled together). Three API calls / one-hour integration for sellers.
- **Wedge / entry point:** Buying GPUs and other infrastructure for AI workloads.

## Market & Competition

- **Market size:**
- **Key competitors:**
  - Skyfire — pivoted to KYC, focused on stable coins only
  - Nakuta — targeting consumer use cases (airlines, shopping)
  - Stripe / PayPal — serve human flows; different channel segmentation
- **Differentiation:**
  - Agent-native vs agent-assisted: Inflowpay registers the agent itself, pays, and purchases entire commerce flow (vs. existing accounts where the agent merely helps with tasks)
  - Policy engine no other payment wallet has today — authorized-spender model where the agent operates under guardrails and reports back to a human
