## Business Description

- **Problem:** AI has decentralized enterprise development and creation; security teams need to keep pace without slowing innovation
- **Product:** Endpoint security for AI tools — covers AI browsers, coding agents, note-takers, Lovable-type apps, plus their extensions, plugins, MCP, and user-built artifacts
- **Wedge:** Endpoint workspace coverage (vs. cloud-only AI security platforms); detects malicious extensions and weekly active attacks on customer endpoints

## Market & Competition

- AI security has three pillars per founder framing:
  1. Prompt injection / DLP (older generation)
  2. AI agent governance (Witness AI, Noma)
  3. AI endpoint security ← Pluto's focus
- Differentiation from Witness AI:
  - Witness: network-level monitoring, agent compliance, data access control
  - Pluto: endpoint application security, extension/plugin management, malicious artifact detection
- Some feature overlap, but core problems differ
- Other competitive references: CrowdStrike / Defender (limited AI tool coverage), Koi (acquired), 6–7 stealth startups
- ~2 employees on Harmonic (as of 2026-04-28); LinkedIn followers +31% over 90 days

## GTM & Business Model

- Product covers three security layers:
  1. **Tools** — AI browsers, coding agents, note-takers, Lovable-type applications
  2. **Ecosystem** — extensions, plugins, skills, MCP, third-party packages
  3. **User-created assets** — applications, agents, workflows with unclear production status
- Coverage spans both locally installed and SaaS-based AI tools
- Customer segmentation:
  - **Enablers (~80%)** — CEO/board pressure to adopt AI; security scrambling to keep up
  - **Blockers (~20%)** — restrictive, tool-by-tool approval policies
- Approach: enable maximum use cases with security controls and risk assessment
