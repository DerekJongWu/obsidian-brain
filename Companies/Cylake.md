## Business Description

- **Problem:** Most modern security analytics platforms (SIEM, XDR, data lakes) are SaaS / public-cloud-first. For organizations subject to data sovereignty, Cloud Act risk, or local data-residency requirements (financial services, regulated multinationals with global subsidiaries), pushing raw security telemetry into a US-headquartered SaaS is a non-starter or significant compliance burden.
- **Product:** Hardware appliance + analytics / data-lake platform deployed in the customer's data center or colo facility. Hooks into cloud and SaaS for telemetry collection, but the core analytics + data lake stay on-prem so the customer retains sovereignty over the data and the analytics layer.
- **Wedge / entry point:** Data-sovereignty-sensitive enterprises (Cloud-Act-exposed, multinational with local data-residency requirements, data-center / colo footprint) that have been forced to compromise on SaaS security analytics.

## Market & Competition

- **Market:** AI-native security analytics / XDR / data-lake-for-security — overlapping with SaaS-first incumbents (Palo Alto Networks XDR, Splunk, SentinelOne, CrowdStrike, Snowflake-for-security) but verticalized to on-prem / sovereign deployments.
- **Differentiation:**
  - On-prem appliance + data-lake architecture vs. SaaS-first incumbents — directly addresses Cloud Act / sovereignty concerns
  - Founder-level credibility in network and endpoint security (Nir Zuk, Udi Shamir, Wilson) lets them recruit CISO-level design partners cold
  - Recruiting senior PAN alumni across product (Nick Campagna VP PM, Maya Marcus CPO, René Bonvanie CMO, Danelle Au VP PMM) — execution / GTM bench stacked from the start
